# Contributing to SIGIL

Thank you for your interest in contributing to the SIGIL protocol!

## How to Contribute

### Specification

- Open an issue to propose changes to the spec
- Submit PRs against `spec/` documents with clear rationale

### Rust Crate (`sigil-rs`)

- Fork the repository
- Create a feature branch
- Ensure `cargo test` passes
- Submit a pull request

### Reporting Issues

- Use GitHub Issues for bug reports and feature requests
- For security vulnerabilities, please email <security@sigil-protocol.org>

## Code of Conduct

Be respectful. We follow the [Rust Code of Conduct](https://www.rust-lang.org/policies/code-of-conduct).
